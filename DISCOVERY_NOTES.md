## Extracting

> Extraction involves obtaining the raw good from its source ([[Manufacturing processes]])

#### Procycling Stats

These guys have an actual API if you're willing to pay (I'm not) and there's a [scraper tool that someone created to extract the data from the pages of the site](https://github.com/themm1/procyclingstats). The scraper seems to be maintained though it's noted as error-prone. Might give it a go :shrug:

https://www.procyclingstats.com/races.php?popular=pro_me&s=upcoming-races&category=1
https://www.procyclingstats.com/races.php?s=races-database&name=&nation=&class=1.UWT&category=1&year=&season=2026&month=&filter=Filter

#### UCI

You can get a nice JSON response from the calendar endpoint with filters for road races (`discipline=ROA`) in the Men's Elite (`raceCategory=ME`) for the current season (2026 → `1056`).

```http
GET https://www.uci.org/api/calendar/upcoming?discipline=ROA&raceCategory=ME&seasonId=1056
```

Response hierarchy-ish:

```plaintext
.
└── items[] # months
    ├── month
    ├── year
    ├── monthName
    ├── isCurrentMonth
    └── items[] # days
        ├── day
        ├── month
        ├── year
        ├── competitionDate
        └── items[] # races
            ├── name
            ├── colourCode
            ├── venue
            ├── country
            ├── continentCode
            ├── dates
            ├── detailsLink
            │   ├── title
            │   ├── url
            │   └── isExternal
            └── isUciEvent
```

No direct API that I could find, but could be scraped.

```http
GET https://www.domestiquecycling.com/:locale/cycling-races/:race-name/:year/startlist/
```

Examples

```http
GET https://www.domestiquecycling.com/en/cycling-races/ronde-van-vlaanderen/2026/startlist/
```
